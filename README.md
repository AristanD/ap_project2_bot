# Проект 2. Телеграм-бот

## Описание

Реализация проекта 2 по предмету Прикладной Python от Д.В. Васильева. Проект выполнен полностью согласно ТЗ и архитектурно соответствует принципам SOLID:

1. Начало работы с выводом основной информации
2. Настройка профиля пользователя
3. Логгирование воды, еды, времени и типа активности
4. Расчёты дневных норм согласно ТЗ (нормы воды и калорий)
5. Для отслеживания норм воды был подключён Open Weather Map API (аналогично проекту 1)
6. Для отслеживания каллорий на 1 грамм еды был подключен Open Food Facts (API-ключ не требуется)
7. Для хранения данных конкретного пользователя была подключена БД SQL LITE (данные хранятся в самом репозитории)

## Функционал

### Настройка профиля пользователя

Команда:

```
/set_profile
```

Запрашивается всё инфо о пользователе согласно ТЗ (вес, рост, возраст, кол-во активности в день в минутах, город проживания и сколько калорий в день пользователь хочет тратить). Вопросы задаются последовательно, ответы пользователя предполагаются после каждого вопроса

После этого рассчитываются:

* дневная норма калорий
* дневная норма воды (с учётом температуры)

### Логирование воды

```
/log_water <мл>
```

Пример:

```
/log_water 250
```

Кол-во выпитой воды сохраняется в отдельную таблицу БД, прогресс по норме в день соответственно обновляется

### Логирование еды

```
/log_food <название продукта>
```

Пример:

```
/log_food Паста болоньезе
```

Используется OpenFoodFacts API для получения калорийности продукта после чего идёт вопрос от бота о массе съеденного продукта/блюда.

### Логирование тренировок

```
/log_workout <тип> <минуты>
```

Пример:

```
/log_workout running 30
```

Рассчитывается:

* сожжённые калории
* дополнительную норму воды

Доступные виды активностей:

* running - 10 ккал в минуту (кардиотренировка/бег)
* cycling - 8 ккал в минуту (вело)
* swimming - 9 ккал в минуту (плавание)
* walking - 4 ккал в минуту (ходьба/прогулка)
* gym - 6 ккал в минуту (силовая тренировка)

### Просмотр прогресса

```
/check_progress
```

Пользователю выводится:

* Сколько воды было выпито за день
* Сколько калорий было употреблено
* Сколько калорий было израсходовано
* Сколько осталось (баланс)

## Схема БД SQLite

### users

| поле         | тип         |
| ---------------- | -------------- |
| id               | INTEGER PK     |
| telegram_id      | INTEGER UNIQUE |
| weight           | INTEGER        |
| height           | INTEGER        |
| age              | INTEGER        |
| activity_minutes | INTEGER        |
| city             | TEXT           |
| calorie_goal     | INTEGER        |
| water_goal       | INTEGER        |

### water_logs

| поле   | тип     |
| ---------- | ---------- |
| id         | INTEGER PK |
| user_id    | FK users   |
| amount_ml  | INTEGER    |
| created_at | DATETIME   |

### food_logs

| поле     | тип     |
| ------------ | ---------- |
| id           | INTEGER PK |
| user_id      | FK users   |
| product_name | TEXT       |
| grams        | INTEGER    |
| calories     | FLOAT      |
| created_at   | DATETIME   |

### workout_logs

| поле        | тип     |
| --------------- | ---------- |
| id              | INTEGER PK |
| user_id         | FK users   |
| workout_type    | TEXT       |
| minutes         | INTEGER    |
| calories_burned | INTEGER    |
| created_at      | DATETIME   |
